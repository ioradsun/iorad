

# Add Story Assets Tab (Loom + iorad Tutorial) to Company Dashboard

## Overview

Add a new **Story Assets** section to the company detail page that displays a Loom script (narrative layer) and an iorad tutorial configuration (mechanism layer), both generated by the existing `generate-cards` edge function. This creates the escalation chain: Signals → Story → Loom → iorad Tutorial → Outreach.

## What Changes

### 1. Update the generate-cards edge function prompt

Extend the output JSON schema in the `DEFAULT_SYSTEM_PROMPT` (and the `cards_prompt_template` in `ai_config`) to include a new `story_assets` object inside `assets`:

```text
"assets": {
  "email_sequence": { ... },
  "linkedin_sequence": [ ... ],
  "story_assets": {
    "active_strategy": "",
    "primary_asset": {
      "type": "loom",
      "title": "",
      "purpose": "",
      "covers": [],
      "when_to_send": "",
      "intro_message": "",
      "loom_script": ""
    },
    "supporting_asset": {
      "type": "iorad_tutorial",
      "title": "",
      "environment": "",
      "what_it_guides": [],
      "business_outcome": "",
      "when_to_send": "",
      "intro_message": "",
      "embed_context": ""
    }
  }
}
```

Add the Story Assets generation rules from your prompt into the system prompt:
- Loom script structure (opening hook, friction walkthrough, what breaks, controlled execution, soft invitation)
- iorad tutorial fields (environment, what_it_guides, business_outcome, when_to_send, intro_message, embed_context)
- Critical rules (no product pitch in Loom, no marketing in tutorial, align to one strategy, execution-focused)

### 2. Update CompanyDetail.tsx -- new Story Assets UI section

Add a new section between the Dashboard Cards and Outreach Assets sections:

```text
+==================================================+
|                                                    |
|  STORY ASSETS                                      |
|  Active Strategy: [strategy name]                  |
|                                                    |
|  ┌─────────────────────────────────┐               |
|  │  Loom Script (Narrative Layer)  │  [Ready: Y/N] |
|  │  Title / Purpose                │               |
|  │  Covers: [bullet list]         │               |
|  │  When to send: [timing]        │               |
|  │  Intro message: [copyable]     │               |
|  │  ── Script ──                  │               |
|  │  [full loom script, copyable]  │               |
|  └─────────────────────────────────┘               |
|                                                    |
|  ┌─────────────────────────────────┐               |
|  │  iorad Tutorial (Mechanism)    │  [Ready: Y/N]  |
|  │  Title / Environment           │               |
|  │  What it guides: [bullets]     │               |
|  │  Business outcome              │               |
|  │  When to send / Intro message  │               |
|  │  Embed context                 │               |
|  └─────────────────────────────────┘               |
|                                                    |
+==================================================+
```

Each asset card includes:
- A readiness badge (checks if key fields are populated)
- Copy buttons on the intro message and loom script
- Clean, scannable layout matching the existing card design

### 3. Types and parsing

Add TypeScript interfaces for the new story_assets shape. Update the `assets` parsing logic to extract `story_assets` alongside `email_sequence` and `linkedin_sequence`.

## Technical Details

### Files to modify

| File | Change |
|------|--------|
| `supabase/functions/generate-cards/index.ts` | Add story_assets to the DEFAULT_SYSTEM_PROMPT schema + generation rules |
| `src/pages/CompanyDetail.tsx` | Add StoryAssetsUI component, types, and render between cards and outreach |

### No database changes needed
The `story_assets` object is stored inside the existing `assets_json` JSONB column in `company_cards`. No migration required.

### Re-generation required
Existing cards will need to be re-generated (click "Generate Cards" again) to produce the new `story_assets` data. The UI will gracefully hide the section if the data isn't present yet.
